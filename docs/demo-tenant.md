# Демо / sandbox-тенант для продаж

Источник: Founder's Playbook, Scale/GTM — *«a well-built demo environment closes deals while you're
in board meetings»*. Цель — чтобы лид/инвестор за минуту увидел работающий продукт (Лия отвечает,
витрина, оплата, наполненная воронка в панели), без ручного онбординга и без риска для боевых тенантов.

## Что это
Изолированный тенант `slug='demo-sandbox'` с:
- включённой **Лией** (свой демо-промпт «Демо-Академия», `ai_enabled=1`),
- 2 товарами (демо-курс 4900 ₽, консультация 2900 ₽),
- наполненной **воронкой**: 3 лида (new / nurturing / converted) + сообщения + 1 оплаченный заказ —
  чтобы панель сразу показывала живую картину (а не пустые экраны).

## Как сеять (безопасно)
`scripts/seed_demo_tenant.py` — idempotent, реверсивный, с **гардом прода**:
```bash
# DEV-пруф (по умолчанию; на боевой risuy без флага — ОТКАЗ):
SEED_DSN="postgresql://gen_user:<pw>@81.31.246.136:5432/risuy_dev?sslmode=require" \
    ./.venv-smoke/bin/python scripts/seed_demo_tenant.py
# удалить:
SEED_DSN="...risuy_dev..." ./.venv-smoke/bin/python scripts/seed_demo_tenant.py --teardown
```
Гарантии: трогает ТОЛЬКО `demo-sandbox` (другие тенанты — никогда), всё в одной транзакции,
повторный запуск пересоздаёт начисто. Пароль owner-DSN — inline из Timeweb API.

**Проверено на risuy_dev (2026-06-18):** создан (2 товара / 3 лида / 1 оплата / 3 настройки Лии),
идемпотентность ✅, гард прода ✅; `metrics_report` на dev показал воронку (конверсия 33.3%, 4900 ₽).

## Как лид взаимодействует (когда оживёт на проде)
1. Лид пишет демо-боту → **Лия отвечает** (демо-промпт продаёт «Демо-Академию»).
2. Слово «магазин» → витрина из 2 товаров → «Купить» → ссылка оплаты ЮKassa (демо-магазин).
3. Оператор видит диалог в панели «Диалоги», может ответить — полный цикл продукта за минуту.

## Промоут на прод (когда дашь демо-бот-токен)
ДБ-слой готов; для «живого» демо на проде нужен **отдельный демо-бот** (Telegram, опц. VK/MAX):
1. Завести демо-бота у @BotFather → токен (НЕ боевой бот тенантов).
2. Засеять демо-тенант на проде: `SEED_ALLOW_PROD=yes SEED_DSN="...risuy..." …seed_demo_tenant.py`.
3. В панели за активного демо-тенанта подключить токен в «Каналы»/«Ключи» (vault) → канал оживёт ≤60с.
4. (Опц.) демо-магазин ЮKassa в «Продукты»→«Приём оплаты», чтобы «Купить» вёл на реальную тест-оплату.
5. Дать лидам ссылку на демо-бота (в КП, на лендинге info.pro-agent-ai.ru, в outbound).

## Дальше (self-serve демо, отдельная волна — по сигналу)
- Кнопка «Попробовать» на лендинге → диплинк в демо-бота с предзаполненным сценарием.
- Авто-сброс демо-данных по расписанию (повторный seed) — чтобы демо всегда свежий.
- Перевод заинтересованного демо-лида в реальный онбординг (снимает узкое горлышко из `bottleneck-map.md`).
