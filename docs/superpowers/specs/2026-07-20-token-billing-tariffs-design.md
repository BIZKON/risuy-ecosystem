# Спека: Токен-биллинг и тарифы «ИИ-Агент Про»

**Дата:** 2026-07-20 · **Статус:** дизайн (ждёт ревью владельца) · **Владелец:** @profysales
**Тип:** смена биллинг-модели (бэкенд-панель + оферта + кабинет + лендинг)

---

## 1. Цель и мотивация

Перейти с текущей модели «квота сообщений + постоплата овереджа» на **единую токен-модель по образцу ElevenLabs**: один пул токенов на ВСЁ, пополняемый ₽-кошелёк на сверх-пул, всё включено в один тариф.

**Зачем:**
- **Конверсия** — одна покупка, пользуешься всем; без à-la-carte «доплати за каждую опцию» (владелец: à-la-carte режет продажи).
- **Прозрачная маржа** — видно в ₽ и % на каждом действии и тарифе.
- **Закрыть утечки** — сейчас ~11 ресурсоёмких путей не списываются с клиента (эмбеддинги/RAG на каждый ответ, DaData, суммаризация памяти и др.).
- **Гибкость** — новые опции (голос, маркетинг-штаб) подключаются как новые строки прайса из того же пула, без переделки биллинга.

## 2. Модель

- **Единица — токен.** Единый пул на всё: чат-ответы + все опции.
- **Тариф = месячный пул токенов, всё включено.** Дорогие модули (голос, маркетинг-штаб) разблокированы на старших тарифах, но тратят тот же пул.
- **Двойной прайс на действие:** X токенов (из пула) / Y ₽ (из кошелька сверх пула). **Курс: 1 токен = 0,0015 ₽.**
- **₽-кошелёк** пополняемый любой суммой (без перепокупки тарифа). **Авто-фолбэк:** пока есть токены — списываем токены; кончились — списываем ₽ из кошелька по тем же ставкам. **Хард-стоп** при balance ≤ 0 (агент на паузе).
- **UX:** два баланса показываем как единый «запас» + разбивка расхода по разделам (EL-стиль).

## 5. Прайс за действие + маржа (подтверждено)

Наценки: **LLM ×4,26** (вшита в курс 0,0015 ₽/ток) · **DaData ×3** · **голос ×2**.

| Действие | Себест. ₽ | Токен-ставка | ₽ сверх пула | Наценка | Маржа ₽ | Маржа % |
|---|--:|--:|--:|:--:|--:|--:|
| Чат-ответ (~5000 ток) | 1,76 | 5 000 | 7,50 | ×4,26 | 5,74 | 76,5 % |
| 🔜 Маркетинг-пост (~8000 ток) | 2,82 | 8 000 | 12,00 | ×4,26 | 9,18 | 76,5 % |
| Проверка контрагента (DaData) | 7,50 | 15 000 | 22,50 | ×3 | 15,00 | 66,7 % |
| Поиск клиентов (DaData) | 7,50 | 15 000 | 22,50 | ×3 | 15,00 | 66,7 % |
| 🔜 Голос-минута (EL) | 9,00 | 12 000 | 18,00 | ×2 | 9,00 | 50 % |
| 🔜 Аренда номера/мес | 420 | — | 420–590 | pass-through | 0–170 | — |

**Две формулы (критично):**
- **LLM-действия:** токен-ставка = ФАКТИЧЕСКИЙ расход токенов (5000, 8000). НЕ «себест.÷курс» (недобор 4,26×).
- **Не-LLM (DaData/голос):** токен-ставка = себест. × наценка ÷ курс (округлять ВВЕРХ). НЕ break-even.

**Маржа на пул:** Эконом/Стартовый (чат + DaData) — **66,7–76,5 %**; Профи (+голос) — пол **50 %** (если голос-тяжёлый). Пол задаётся наценкой самого дешёвого-по-марже доступного действия.

## 7. Что уже построено vs что менять (миграция)

**Уже в проде (Wave 0–4), переиспользуем:**
- `credit_wallets` (µRUB) + append-only `usage_ledger` + `charge_usage()` (идемпотентность, FOR UPDATE, округление вверх).
- Реальный замер токенов LLM (`metering_worker.py` снапшоты cloud-ai + `ai.py::_capture_gateway_usage`).
- Пополнение: `payments(type='topup')` + `POST /wallet/topup` (ЮKassa) + вебхук `mark_topup_succeeded`.
- Хард-стоп `ai_wallet_blocked` при balance ≤ 0.

**Ключевая работа — СХЛОПНУТЬ два параллельных прод-счётчика одного расхода** (публичная квота «сообщений Лии» `count_ai_messages` vs кошелёк µRUB) в один пул:
1. Ввести слой курса token↔₽ поверх µRUB (не смешивать 3 «токена»: продаваемый / закупочный LLM / µRUB-себестоимость).
2. Выпилить `billing_mode='per_message'` (иначе латентное двойное списание), планы → `cost_multiplier`.
3. Депрекейтить квоту-в-сообщениях + `service_invoices.quota/overage` (старые счета в «сообщениях» несовместимы с токенами → бэкфилл/заморозка).
4. **Починить `/service/subscribe`**: платёж идёт `kind='service_landing'` БЕЗ `tenant_id` → вебхук no-op → кошелёк не пополняется. Прокинуть tenant_id.
5. **Закрыть неучтённые утечки** (тарифицировать в токенах): эмбеддинги+RAG на каждый ответ, индексация БЗ, суммаризация памяти, DaData. Модель без строки в `model_prices` = токены бесплатны → добить прайс.
6. Переписать **оферту** (п.1.6 «Сообщение ИИ» + п.4 квоты 500/1500 + постоплата овереджа → prepaid токен-пул).

## 10. Открытые решения (до реализации)

1. **Калибровка «токенов на ответ»** — снять фактический средний из `usage_ledger` на боевой БЗ (V4 Pro Thinking жжёт больше 5000). Влияет на размер пулов.
2. **Фактическая себестоимость DaData** — 7,5 ₽ (владелец) vs ≈0 free-tier. Если ≈0 — маржа проверки/поиска взлетает до ~93 %.
3. **Токены: сгорают/переносятся** + **возврат аванса** — юр-условия для оферты.
4. **Наценка голоса** — поднять ×2→×2,5–3 (цена 22–27 ₽/мин), чтобы пол маржи Профи был выше 50 %.
5. **Маркетинг-штаб (позже):** Вордстат API vs парсинг; Instagram в автопостинге (геоблок РФ); блог на поддомене у нас vs у клиента.

## 11. Риски

- **Маржа:** разнородная себестоимость (голос ≫ чат ≫ DaData); пол 50 % на голосе — крутится наценкой.
- **Юридический:** переписать оферту под prepaid токен-пул + смену единицы; условия сгорания/возврата аванса.
- **FX:** USD-вендоры (EL) → буфер +15–20 %, помесячная переоценка.
- **Миграция:** схлопывание двух прод-счётчиков; бэкфилл/заморозка старых счетов в «сообщениях»; не сломать Школу Лесова (переходник без плана).

## 12. Границы (non-goals)

- Внутренности голосовой телефонии (номера/CRM/Voximplant-обвязка) — отдельная спека.
- Внутренности Маркетинг-штаба (Вордстат/блог/автопостинг) — отдельная спека.
- НЕ строим новый кошелёк — переиспользуем `credit_wallets`/`usage_ledger`.

## 13. Критерии приёмки

- Один пул токенов, один счётчик расхода (старая квота «сообщений» депрекейтнута, двойного списания нет).
- Кабинет показывает единый «запас» + разбивку по разделам; пополнение работает; хард-стоп при 0.
- Ранее неучтённые расходы (эмбеддинги/DaData) списываются из пула.
- Лендинг == бэкенд == оферта (цены/единицы совпадают).
- Маржа наблюдаема и не ниже заявленных полов (66,7 % / 50 %).
