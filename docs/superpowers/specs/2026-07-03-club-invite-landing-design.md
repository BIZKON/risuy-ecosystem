# Спека: Публичное приглашение в клуб (лендинг + ссылка-карточка)

**Дата:** 2026-07-03 · **Владелец:** @profysales · **Статус:** дизайн утверждён, ждёт ревью спеки.

## 1. Цель и контекст

Легальный входящий рычаг роста клуба: публичная (без авторизации) страница-лендинг клуба
тенанта с кнопкой «Вступить» → deep-link в бот-воронку клуба (152-ФЗ-совместимую). Оператор
берёт шейрабельную ссылку в панели и распространяет её в **своих** каналах/сообществе/подписи.

**Почему это легально (в отличие от парсинга/холодного контакта):** лендинг — публичная
страница, на которую пользователь **приходит сам** (не рассылка по сетям электросвязи → вне
стены ФЗ-38 ст.18). Вступление проходит через существующий флоу согласия (152-ФЗ ст.9). Ни
одного холодного контакта — соответствует красной линии клуба (спека клуба §1).

**Не-цель (вне scope):** батч-приглашение своих лидов, реферальный луп, карта пробелов
(отдельные рычаги, очередь после этого); редактируемый оператором слоган (решение владельца —
лендинг минимальный авто-генерируемый); клуб-UX фильтры/CSV/дашборд (отдельная спека, в очереди).

## 2. Решения владельца (зафиксированы)

- Лендинг — **минимальный авто-генерируемый**: название клуба (из реквизитов оператора) +
  генерик-текст ценности + кнопка. Без редактируемого слогана → **новых настроек/DDL нет**.
- Ссылку оператору показываем **отдельной карточкой в `/club`** (не в «Лид-магните»).

## 3. Компоненты

### 3.1. Публичный лендинг бота — `GET /club/{slug}`
Живёт в `bot-telegram/bot.py` рядом с существующим `_legal_page` (`/legal/{slug}/{doc_type}`),
на том же публичном aiohttp-сервере бота (база = `config.BOT_PUBLIC_BASE_URL`).

- `_club_landing(request)`:
  - `slug` из `match_info`.
  - `data = await db.get_club_landing_data(slug)`.
  - `None` → HTTP 404 «Клуб не найден или не настроен» (не рекламируем клуб, который не может
    принять вступление).
  - иначе → `web.Response(_club_landing_html(data), content_type="text/html")` +
    `Cache-Control: public, max-age=600` + `X-Content-Type-Options: nosniff` (как `_legal_page`).
- `_club_landing_html(data) -> str` — чистый самодостаточный HTML (без внешних ресурсов):
  заголовок = название клуба (`«Клуб предпринимателей {operator_name}»`), генерик-абзац ценности
  («находите комплементарных партнёров; знакомства — только по взаимному согласию»), краткая
  строка «что собираем: данные вашего бизнеса; подробнее — Политика» со ссылкой на
  `policy_url`, большая кнопка-ссылка **«Вступить в клуб»** → `deeplink`. Тон — деловой,
  честный, без давления.

### 3.2. Данные лендинга — `get_club_landing_data(slug)` (bot `db.py`)
**Переиспользует `get_legal_doc_data(slug)`** — он уже делает lookup по слагу (join `tenants`,
owner-DSN обходит RLS), **гейтит готовность** (возвращает `None`, если `operator_name`/
`operator_inn`/`operator_email` не заполнены — те же реквизиты, что требует `_club_start`-гард
воронки) и отдаёт `operator_name`. Обёртка добавляет:
- `policy_url` = `{BOT_PUBLIC_BASE_URL}/legal/{slug}/privacy` (как `legal_privacy_url` в
  `get_funnel_config`; если база не задана — ссылку на Политику не показываем, лендинг остаётся).
- `bot_username` — из того же источника, что промоушен-флоу (кнопка «Пригласить в клуб»:
  `runtime_status`/`_bot_username()`). `deeplink = "https://t.me/{bot_username}?start=club"`.
  Если `bot_username` пуст → `None` (без deep-link кнопка бессмысленна → 404).
- Возвращает `{operator_name, deeplink, policy_url}` или `None` (тенант не найден / реквизиты не
  заполнены / нет bot_username).

### 3.3. Карточка-приглашение в панели — `/club` (admin-panel)
В рендере `GET /club` (`admin-panel/app.py`) добавить блок «Ссылка-приглашение в клуб» в
`templates/club.html`:
- **URL лендинга**: `{BOT_PUBLIC_BASE_URL}/club/{slug}` (slug активного тенанта).
- **Deep-link бота**: `https://t.me/{bot_username}?start=club`.
- Кнопки «Копировать» (минимальный inline-JS `navigator.clipboard`) для каждого; текст также
  выделяемый (фолбэк). QR — не в первой версии (YAGNI; можно потом).
- Показываем карточку только если клуб готов принять вступление (реквизиты оператора заполнены);
  иначе — подсказка «Заполните реквизиты оператора в «Лид-магните», чтобы открыть клуб».
- Данные для карточки: `slug` активного тенанта, `BOT_PUBLIC_BASE_URL` (config), `bot_username`
  (`runtime_status`, как промоушен-флоу), флаг готовности (реквизиты).

## 4. Поток данных
`оператор шейрит ссылку` → `посетитель открывает {base}/club/{slug}` → `_club_landing` →
`get_club_landing_data(slug)` → HTML с кнопкой → `t.me/<bot>?start=club` → бот-FSM `_club_start`
(показ согласия 152-ФЗ, гард реквизитов) → вступление + `consent_events(club_join)`.

## 5. Обработка ошибок / краевые случаи
- Тенант по слагу не найден / реквизиты оператора не заполнены → 404 (лендинг) и подсказка
  вместо ссылки (панель). Не отправляем людей в мёртвую воронку.
- `bot_username` пуст (runtime_status ещё не заполнен) → deep-link не строим, показываем
  подсказку; лендинг 404 (без deep-link кнопка бессмысленна).
- Слаг с посторонними символами — роут матчит `{slug}`, `get_club_landing_data` параметризован
  (SQL-инъекции нет); неизвестный слаг → None → 404.
- Кэш лендинга `max-age=600` — приемлемо (реквизиты меняются редко).

## 6. Тестирование
- **unit** (`scripts/club_landing_smoke.py`, PYTHONPATH=bot-telegram): `_club_landing_html`
  содержит deep-link, ссылку на Политику, кнопку «Вступить», название оператора; экранирование
  operator_name (нет HTML-инъекции).
- **db-смоук** (risuy_dev, контроллер inline): `get_club_landing_data` → None для неизвестного
  слага и для тенанта без реквизитов; корректные поля для настроенного тенанта; tenant-скоуп.
- **render-смоук панели** (`admin-panel`): `/club` показывает карточку-приглашение с верным
  URL/deep-link при готовом клубе; подсказку — при незаполненных реквизитах.

## 7. Затрагиваемые файлы (ожидаемо)
`bot-telegram/bot.py` (роут + `_club_landing` + `_club_landing_html`), `bot-telegram/db.py`
(`get_club_landing_data`), `admin-panel/app.py` (данные карточки в `/club`),
`admin-panel/templates/club.html` (блок карточки), новый `scripts/club_landing_smoke.py`.
**Без прод-DDL, без новых настроек.**

## 8. Юр-инварианты (незыблемо)
- Лендинг — публичная inbound-страница; вступление = согласие; оператор шейрит в СВОЕЙ
  аудитории. Холодного контакта / внешнего парсинга нет.
- Лендинг честный, без давления/обещаний; ссылка на Политику обязательна.
- Не является юр-заключением; тексты согласия по-прежнему выверяет юрист (см. досье
  `docs/152fz-club-legal-gate-dossier.md`) до боевого включения тенантам.
