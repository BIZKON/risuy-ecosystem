# SCOPE — risuy-ecosystem («ИИ-Агент Про»)

Контракт границ продукта. Источник: Founder's Playbook, MVP/Launch — *«scope creep without
friction is one of the defining failure modes of AI-era MVPs»*. С агентным кодингом фича стоит
«вечер», поэтому пропал естественный тормоз (цена инженерного времени). Этот файл — замена тормоза:
решение смещается с «можем ли построить?» на **«достаточно ли пользователей сказали, что без этого
не получают ценности?»**.

> Правило: новая фича добавляется ТОЛЬКО если проходит «Критерии добавления» (ниже). Иначе —
> в `docs/handoffs` как идея, не в код. «Легко построить» — НЕ причина строить.

---

## Что это за продукт (одно предложение)

Мульти-тенант **реселлинг-платформа**: каждый клиент-тенант запускает своего ИИ-агента продаж
(**Лия**) в Telegram / VK / MAX — агент ведёт лида по воронке (разговор → триггеры → эскалация
менеджеру → продажа через ЮKassa-кассу тенанта), а оператор может вмешаться из панели.

---

## ✅ Что продукт ДЕЛАЕТ (в проде, граница v1)

- **Разговорный ИИ-агент (Лия)** в Telegram, VK, MAX — единый «мозг» (`ai.ask_ai`), канал-агностично.
- **Воронка лида:** приём → guide → nurture-фоллоуапы → эскалация менеджеру (Слой A) → продажа.
- **Триггеры** (Слой B): 4 типа (стоп-слова / счётчик / документы / намерение), per-tenant.
- **Каналы** (Слой C): self-serve подключение TG/VK/MAX клиентом через «Каналы»/«Ключи» (vault).
- **Продажи:** кнопка «Купить» во всех 3 каналах → витрина → оплата на кассу-СОЗДАТЕЛЯ.
- **Касса клиента (ЮKassa):** каждый тенант принимает оплату за свои товары на свой магазин.
- **«Диалоги»-композер (C3):** оператор отвечает лиду текстом/фото/файлом в VK/MAX из панели.
- **Рассылки** (C3): TG/VK/MAX, текст+медиа, аудитория с фильтрами, отписка словом «СТОП» (152-ФЗ).
- **Мульти-тенантность:** RLS deny-by-default, per-tenant vault/настройки/промпт Лии (white-label).
- **Метеринг и платформенные подписки** тенантов (usage_ledger / service_invoices / subscriptions).

## ⛔ Что продукт НАМЕРЕННО НЕ делает (v1)

Каждый пункт — осознанное «не сейчас», НЕ забытое. Снять — только через «Критерии добавления».

- **НЕ голосовые звонки / обзвон.** Голос приходит как файл/документ, нативного голос-пузыря нет.
  (Холодный голос-обзвон в РФ — отдельный риск-класс, см. навык sales-dept-automation-roadmap.)
- **НЕ полноценная CRM.** Лиды/диалоги/заказы — да; сделки-пайплайны, отчётность как у CRM — нет.
- **НЕ маркетинг-сьют.** Рассылки есть; A/B контента, лид-скоринг ML, мульти-тач атрибуция — нет.
- **НЕ конструктор ботов.** Промпт Лии + триггеры настраиваются; визуального flow-билдера нет.
- **НЕ per-channel триггеры.** Колонка `tenant_triggers.channel` есть, но НЕ подключена — триггеры
  тенант-широкие (один набор на все каналы). Намеренно (дизайн §10 п.5).
- **НЕ MAX-приём через webhook в проде-нагрузке.** Сейчас long-poll (ок для пилота); webhook —
  перед боевой нагрузкой MAX (дока MAX: long-poll не для прода).
- **НЕ per-tenant чек 54-ФЗ.** Тенант-платёж сейчас без фискального чека (Школа — через свой магазин).
- **НЕ ретрай канальной доставки C3** (fire-and-forget по строке; ретрай — фаза 2) и **НЕ кэш media-id**
  рассылок (upload на каждую отправку).
- **НЕ брендовый нотификатор.** Сейчас тестовый @ConversionConsultant_bot.
- **НЕ self-serve регистрация тенантов.** Заведение тенанта — через платформу (admin).

## 📋 Критерии добавления фичи (gate)

Фича переходит из «идея» в «строим», только если выполнено **ВСЁ**:

1. **Сигнал от пользователей, не энтузиазм:** ≥ N реальных тенантов (или квалифицированных лидов
   на продажу) явно сказали, что без этого НЕ получают ценности / не покупают. Не «было бы круто».
2. **Бьёт по метрике:** ожидаемо двигает конкретную метрику из `metrics-framework.md` (активация,
   конверсия, удержание, выручка) — с гипотезой «на сколько».
3. **Не ломает границы:** не превращает продукт в CRM/маркетинг-сьют/конструктор (см. ⛔ выше);
   если превращает — это смена стратегии, обсуждается отдельно, а не «ещё фича».
4. **Считаемая цена/риск:** оценён риск для денежного/RLS/канального пути; если трогает деньги или
   изоляцию тенантов — требует адверсариального ревью + смоук (как W2/circuit-breaker).

Если хоть один пункт не выполнен → идея в backlog (`docs/handoffs`), не в код.

## Очередь снятия ограничений (приоритет по сигналу, не по лёгкости)

Когда появятся боевые тенанты и сигнал — снимать в этом порядке (пересмотреть по данным):
1. **per-tenant чек 54-ФЗ** — как только тенант реально принимает деньги клиентов (комплаенс, не опционально).
2. **MAX long-poll → webhook** — перед боевой нагрузкой MAX-продаж.
3. **Ретрай канальной доставки + кэш media-id** — когда объём рассылок VK/MAX вырастет.
4. **Брендовый нотификатор** — перед боевым white-label.
5. **per-channel триггеры** — только если тенанты попросят разное поведение по каналам.
