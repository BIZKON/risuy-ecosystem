# Предлагаемые тексты согласий клуба (ЧЕРНОВИК для юриста — не задеплоено)

Приведены к составу эталона `build_consent_text`. Плейсхолдеры {…} подставляются из реквизитов
тенанта (get_funnel_config: operator_name, operator_inn, operator_email, privacy_url/legal_privacy_url).
Финально выверяет юрист оператора. Правки в код НЕ внесены — ждут «да» + юр-подписи.

## Требуемое изменение сигнатуры (shared/club.py)
Сейчас: `build_club_consent_text(kind, operator_name)`.
Предложение: `build_club_consent_text(kind, operator_name, operator_inn="", operator_email="", privacy_hint=False)`
+ в `handlers._club_start` тянуть operator_inn/operator_email из `get_funnel_config` тем же путём, что operator_name, и гардить их непустоту (как сейчас гардится operator_name, handlers.py:129). Без DDL.

## 1. club_join (вступление) — предлагаемый текст

```
Оператор: {operator_name}, ИНН {operator_inn}.

Вступая в Клуб предпринимателей, вы даёте согласие на обработку персональных данных вашего бизнеса:
• Данные: название бизнеса, город, сфера/ОКВЭД, что предлагаете и что ищете, позиция в цепочке поставки.
• Цель: подбор комплементарных партнёров внутри клуба и организация знакомств по взаимному согласию.
• Действия: сбор, хранение, использование для автоматизированного подбора партнёров внутри клуба.
• Срок: до отзыва согласия.
• Хранение — на серверах в России.
• Отозвать согласие можно в любой момент: команда /revoke в боте или письмо на {operator_email}.

Полные условия — в политике обработки данных (кнопка ниже).
Нажимая «Вступить в клуб», вы соглашаетесь на обработку указанных данных.
```
Закрывает: cj-inn, cj-datalist, cj-actions-ai, cj-term, cj-revoke, cj-storage-rf, cj-policy-link, cj-separability, L4-consent-text-no-revoke-mechanism, L2-adv-below-etalon.
⚠️ Перечень «Данные» держать в ОДНОМ источнике с полями FSM (reg-ключи handlers.py:151 / validate_club_registration), чтобы не разъезжался.

## 2. intro (знакомство) — предлагаемый текст (ПОКАЗЫВАЕТСЯ = ХЭШИРУЕТСЯ)

```
Знакомство в Клубе предпринимателей. Оператор: {operator_name}, ИНН {operator_inn}.

🤝 Вам предлагают знакомство. Принимая, вы соглашаетесь — при ВЗАИМНОМ согласии обеих сторон —
передать второму участнику: название бизнеса, город, сферу/ОКВЭД, имя контактного лица и телефон.
Контактами обменяемся, только если согласитесь оба.
• Цель: обсуждение возможного партнёрства.
• Отозвать согласие: команда /revoke или письмо на {operator_email}.
```
Кнопки: [✅ Принять] [✖️ Отклонить].
🔴 КЛЮЧЕВОЕ: `text_hash` считать РОВНО над этой показанной строкой (single-source), не над отдельной
`build_club_consent_text('intro')`, которую субъект не видит.
Закрывает: intro-hash-mismatch, L3-intro-shown-neq-hashed, intro-operator, intro-scope, L3-intro-consent-underdescribes-disclosure, L3-intro-policy-link-absent, intro-term-revoke-policy.
⚠️ Перечень раскрытия синхронизировать с `club_intro_contact_card` (texts.py:273-287): сейчас карточка отдаёт название/город/ОКВЭД/контактное лицо/телефон (ИНН в карточку НЕ рендерится — уточнение верификатора).
